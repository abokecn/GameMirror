Guide

这个主页适合做什么

统一入口

将所有游戏镜像、访问页、公告页和备用网址统一收纳，避免后续扩展时结构混乱。

配置驱动

通过 gamesweb.conf 维护站点数据，前端只负责展示，长期维护成本更低。

适合 GitHub

纯静态结构清晰，便于通过 GitHub 编辑、审查变更、回滚版本，并直接接入 Pages 部署。

可继续升级

后续可加入 JSON 数据源、自动生成游戏卡片、搜索、筛选、状态灯与公告系统。

gamesweb.conf 示例

site_name = "GameMirror"
base_url = "https://gamemirror.org"

[[games]]
name = "KARDS"
slug = "kards"
status = "online"
path = "/games/kards/"
target = "https://www.kards.com/"
type = "reverse-proxy"
description = "KARDS 大陆访问入口"

[[games]]
name = "Example"
slug = "example"
status = "planned"
path = "/games/example/"
target = ""
type = "resource-hub"
description = "未来扩展示例"

说明：Cloudflare Pages 本身更适合静态站点；若要做真正的动态反代逻辑，通常还需要结合 Cloudflare Workers / Functions。当前这套主页结构先把“品牌首页 + 目录结构 + 长期维护框架”搭好。
